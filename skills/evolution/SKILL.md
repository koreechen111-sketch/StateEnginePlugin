# Evolution Skill

进化Skill负责分析执行错误、生成改进规则、更新PROJECT.md和RULE.md，实现**所有Agent共同经验的沉淀**。

## 功能

- **错误分析**：分析失败根因，分类错误类型
- **规则生成**：从错误中提取规则，防止再次发生
- **经验沉淀**：更新PROJECT.md沉淀项目特定经验
- **规则沉淀**：更新RULE.md记录通用规则和最佳实践

## 触发时机

### 1. 自动触发（阶段完成时）

由 `using-e2e` 调度器在以下时机自动调用：

- **每个阶段执行完成后**：REQUIREMENT、DESIGN、TEST_DESIGN、TASK_PLAN、TASK_EXECUTION、SYSTEM_TEST 各阶段执行+Review通过后
- **任务执行完成后**：TASK_EXECUTION 阶段每个任务完成后

### 2. Agent 主动触发

各Agent在以下情况主动调用 evolution：

- **识别到公共经验**：发现对后续阶段有帮助的模式、经验、教训
- **识别到需传递信息**：发现需要让后续Agent知道的重要信息
- **发现错误**：自己犯错或Review发现错误时
- **发现有价值经验**：执行过程中发现有效的解决方案或最佳实践
- **人工触发**：用户说"分析错误"、"优化流程"、"沉淀经验"时

### 沉淀内容标准

#### 需要沉淀到 PROJECT.md（项目特定经验）

- **需求分析阶段**：业务场景模式、用户优先级划分方式、常见需求遗漏点
- **设计阶段**：有效的架构模式、技术选型权衡、数据流设计经验
- **用例设计阶段**：测试场景覆盖经验、边界条件识别模式
- **任务规划阶段**：任务分解经验、依赖关系处理方式
- **任务执行阶段**：常见问题解决方案、TDD实践经验、性能优化技巧
- **系统测试阶段**：测试用例设计经验、缺陷发现模式

#### 需要沉淀到 RULE.md（通用规则/最佳实践）

- 各阶段通用的最佳实践
- 可复用的技术规范
- 代码质量标准
- 评审检查清单模板
- 错误预防规则

## 错误分类


| 错误类型    | 说明       | 处理方式 |
| ----------- | ---------- | -------- |
| REQUIREMENT | 需求不清晰 | 需求补充 |
| DESIGN      | 设计缺陷   | 架构调整 |
| CODE        | 代码问题   | 代码修复 |
| SPEC        | 规格偏差   | 规格修正 |
| QUALITY     | 质量问题   | 质量改进 |
| TOOL        | 工具问题   | 环境修复 |

## 输入

- 当前阶段/任务执行结果
- 阶段产物（requirements/SRS.md, design/design.md 等）
- 当前 PROJECT.md 内容（如果存在）
- 当前 RULE.md 内容（如果存在）
- Review 结果（通过/失败/需修改）

## 输出

- 更新的 PROJECT.md（项目特定经验）
- 更新的 RULE.md（通用规则）
- 进化报告（可选）

## 使用示例

```
用户：任务T003连续失败了2次
你：使用 evolution-skill 分析错误

[分析错误类型]
[生成改进规则]
[更新PROJECT.md]
...

进化完成，建议：
1. 补充T003的规格定义
2. 明确依赖关系
3. 增加前置条件检查
```

## 进化报告格式

```markdown
# Evolution Report

## 时间
2026-03-23T10:00:00Z

## 触发原因
- 任务T003连续失败2次

## 错误分析

### 错误类型
DESIGN

### 错误详情
任务T003的依赖关系不清晰，导致执行顺序错误

### 根因分析
1. 任务分解时未明确标记依赖
2. 执行顺序假设错误

## 改进建议

### 规则1：任务依赖必须明确
在task-planning阶段，每个任务必须明确声明依赖关系

### 规则2：依赖验证
任务执行前必须验证依赖是否完成

## 更新的文件

- PROJECT.md
- AGENT.md（可选）

## 下次遇到类似问题
按照上述规则处理，避免重复犯错
```

## 集成

### 自动触发（由 using-e2e 调度）

被 `using-e2e` Skill 在以下时机自动调用：

- **每个阶段执行完成后**：REQUIREMENT、DESIGN、TEST_DESIGN、TASK_PLAN、TASK_EXECUTION、SYSTEM_TEST 各阶段 Review 通过后
- **任务执行完成后**：TASK_EXECUTION 阶段每个任务通过 Review 后

### 主动触发（由 Agent 调用）

各 Agent 在以下情况主动调用：

- 识别到公共经验或需传递信息时
- 发现错误时
- 人工触发时

## 注意事项

1. **禁止甩锅**：错误分析要客观，重点在改进而非指责
2. **可操作**：每条规则必须有明确的操作指引
3. **有据可查**：规则要有错误证据支持
4. **适度进化**：不要过度优化，每次进化聚焦1-2个问题
